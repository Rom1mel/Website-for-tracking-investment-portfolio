## Цель проекта:
Разработка и внедрение рабочего MVP веб-сервиса C&D Invest, который предоставит пользователям возможность централизованно отслеживать и анализировать гибридные инвестиционные портфели, объединяющие традиционные (акции, ETF) и цифровые (криптовалюты) активы, через интуитивно понятный веб-интерфейс.

## Технологии:
- Frontend: HTML, CSS, JavaScript
- Beckend: Python, Jango
- Data base: 
- Tools: GitHub

## Цели на семетр:
1. Реализовать регистрацию и вход (отдельный ЛК для каждого пользователя)

Backend:
 - Создать модель пользователя (User) в БД (id, email/login, password_hash, created_at).
 - Настроить подключение к базе данных.
 - Реализовать хеширование пароля (bcrypt / argon2).
 - Создать endpoint:
 - POST /register
 - POST /login
 - Добавить проверку:
 - существует ли пользователь
 - корректность пароля
 - Реализовать сессии или JWT-аутентификацию.
 - Настроить middleware для проверки авторизации.
 - Ограничить доступ к страницам активов только для авторизованных пользователей.

Frontend:
 - Создать HTML-страницу регистрации.
 - Создать HTML-страницу входа.
 - Добавить формы (input: email, password).
 - Добавить валидацию (required, minlength).
 - Подключить JS для отправки формы (fetch / axios).
 - Обрабатывать ошибки (неверный пароль, пользователь существует).
 - Сделать редирект после успешного входа.

2. Создать базу данных для хранения активов
 - Выбрать БД (PostgreSQL / MySQL / SQLite).
 - Создать таблицу users.
 - Созить таблицу assets:
 - id
 - user_id (внешний ключ)
 - asset_name
 - quantity
 - purchase_price
 - purchase_date
 - created_at
 - Настроить связь user → assets (one-to-many).
 - Проверить, что активы одного пользователя не видны другому.
 - Настроить миграции (если используется ORM).
 - Протестировать сохранение данных после перезапуска сервера.

⸻

3. Научить сервер принимать и сохранять новые активы
 - Создать endpoint POST /assets.
 - Добавить проверку авторизации.
 - Реализовать валидацию входных данных:
 - asset_name не пустой
 - quantity > 0
 - price > 0
 - Сохранять актив в БД с привязкой к user_id.
 - Вернуть ответ об успешном создании.
 - Обработать возможные ошибки (400, 500).

⸻

4. Сделать заглушку главной страницы
 - Создать HTML-страницу index.html.
 - Добавить базовую структуру:
 - header
 - блок списка активов
 - кнопка «Добавить актив»
 - кнопка «Выйти»
 - Подключить CSS-стили.
 - Добавить проверку авторизации при загрузке страницы.
 - Настроить редирект:
 - неавторизованный → страница входа
 - авторизованный → главная страница

⸻

5. Подключить API для получения цен акций/криптовалют
 - Выбрать API (например: Alpha Vantage, Binance API и т.д.).
 - Зарегистрироваться и получить API-ключ.
 - Создать сервис/модуль для работы с API.
 - Реализовать GET-запрос к API.
 - Обработать:
 - успешный ответ
 - ошибку
 - лимиты запросов
 - Обновлять текущую цену при отображении активов.
 - Рассчитать:
 - текущую стоимость (quantity × current_price)
 - прибыль/убыток
 - Добавить кеширование (по желанию).

⸻

6. Сделать интерфейс для добавления актива

Frontend:
 - Создать HTML-форму:
 - Название актива
 - Количество
 - Цена покупки
 - Дата покупки
 - Добавить кнопку «Сохранить».
 - Добавить валидацию полей.
 - Подключить JS для отправки данных на POST /assets.
 - Показать сообщение об успешном добавлении.
 - Очистить форму после сохранения.

Backend:
 - Проверить получение данных.
 - Вернуть корректный JSON-ответ.

⸻

7. Отобразить список активов на главной странице

Backend:
 - Создать endpoint GET /assets.
 - Возвращать активы только текущего пользователя.
 - Формировать JSON-ответ со списком активов.

Frontend:
 - При загрузке страницы отправлять запрос к GET /assets.
 - Полученные данные отобразить в:
 - таблице (table)
 - или списке (ul/li)
 - Отобразить:
 - название
 - количество
 - цена покупки
 - текущая цена
 - прибыль/убыток
 - Добавить обновление списка после добавления нового актива.
